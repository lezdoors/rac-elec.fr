Recommandations générales pour réduire le CLS

Pour les images et médias

  • Toujours spécifier width et height - Cela permet au navigateur de réserver l'espace nécessaire.
  • Utiliser l'attribut aspect-ratio - Pour maintenir le rapport hauteur/largeur et éviter les sauts.
  • Implémenter des placeholders - Utiliser des techniques comme le blur-up ou les squelettes de chargement.
  • Éviter les images sans dimensions - Toujours définir des dimensions explicites pour les images.

Pour les polices

  • Utiliser font-display: swap - Pour éviter que le texte ne soit invisible pendant le chargement de la police.
  • Précharger les polices critiques - Avec <link rel="preload" as="font">.
  • Considérer le system font stack - Utiliser des polices système pour les performances.
  • Utiliser size-adjust et font-family fallbacks - Pour minimiser les changements de taille entre les polices de substitution et les polices web.

Pour les annonces et embeds

  • Réserver un espace fixe - Toujours définir une hauteur et une largeur fixes pour les conteneurs d'annonces.
  • Utiliser des dimensions fixes pour les iframes - Définir width et height pour tous les iframes.
  • Injecter le contenu dynamique vers le bas - Éviter d'insérer du contenu au-dessus du contenu existant après le chargement.

Pour le contenu dynamique

  • Préallouer l'espace - Utiliser min-height pour réserver de l'espace pour le contenu qui sera chargé.
  • Utiliser des squelettes de chargement - Montrer une version "squelette" du contenu pendant le chargement.
  • Éviter les transformations qui changent la taille - Préférer transform aux changements de taille pour les animations.
  • Utiliser position: absolute - Pour les éléments qui ne devraient pas affecter le flux de la page.

Techniques de chargement

  • Charger les styles critiques en ligne - Mettre les styles essentiels dans le <head> pour éviter les FOUC.
  • Différer les scripts non critiques - Utiliser async ou defer pour les scripts qui ne sont pas essentiels.
  • Charger le contenu tiers de manière asynchrone - Utiliser des techniques comme l'Intersection Observer pour charger le contenu uniquement lorsqu'il est nécessaire.
